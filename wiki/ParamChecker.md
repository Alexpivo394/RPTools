# ParamChecker

## Введение

**ParamChecker** — это твой цифровой ревизор. Он проверяет модели на соответствие требованиям к параметрам, сравнивает с эталонами LOI (Level of Information) и даёт возможность тонко настроить, кого проверять, по каким параметрам и с какими условиями.

## Цель

Проверить, соответствуют ли элементы модели заданному уровню информации (LOI), путём сравнения с параметрическими правилами.

## Подход

1. Создаём профиль, внутри которого будут действовать правила для всех моделей этого профиля. Например, удобно создавать по профилю на раздел.
2. Добавляем в профиль модели, которые нужно проверить.
3. Задаём, **по каким параметрам** и **какие условия** должны выполняться.
4. Плагин обходит модель, проверяет все элементы и… либо выдаёт список счастливчиков, либо показывает список нарушителей и экспортирует отчёт в Excel.

2. **Боковое меню навигации** — конфигурации, настройки, а также кнопка добавления конфигураций. Чтобы перейти на нужную страницу, нажмите на неё в этом меню.

## Окно настройки профиля

<img width="820" alt="Окно настройки профиля" src="https://github.com/user-attachments/assets/f0a3ea83-c1e9-4662-9602-29e0b1ea9714" />

Окно настройки профиля разделено на 2 области: настройка моделей и настройка правил экспорта. В каждой области внизу есть кнопка **«Добавить»**, которая добавляет модель или правило.

> [!TIP]
> У каждого профиля слева от имени есть чекбокс. Если он не активен, профиль будет проигнорирован и не попадёт в отчёт.

> [!TIP]
> Чтобы переименовать или удалить профиль, нажмите по нему правой кнопкой мыши и выберите нужную функцию в контекстном меню.

### Модели для экспорта

Каждое поле с моделью содержит 3 текстовых поля и кнопку «Удалить».

- **Путь к модели Revit** — путь к модели на сервере, которую нужно проверить. Например: `RSN:\\vsrevit00\Projects\42_Pioner High Life\IOS\EOM\HL_EOM_K4_R22.rvt`
- **Имя вида** — имя вида в модели, на котором плагин будет искать элементы.
- **Ключевое слово РН** — символ или слово, с которого начинаются рабочие наборы, которые необходимо закрыть при открытии модели.

### Настройка правил экспорта

Каждое поле с правилом содержит 3 текстовых поля. У 2 полей есть кнопка вызова окна настройки этого поля.

- **Название** — название правила. Вкладка в Excel с элементами из этого правила получит это же название.
- **Фильтр** — задаёт фильтр, по которому будут выбраны элементы для проверки в этом правиле.

  > [!WARNING]
  > Вручную задать фильтр с нуля не получится, если только вы не понимаете структуру JSON-настроек. Поле сделано редактируемым, чтобы удобно копировать настройки фильтров между правилами.

  Чтобы задать фильтр, нажмите кнопку с тремя точками справа от поля — откроется [окно настройки фильтра](#окно-настройки-фильтра).

- **Параметры для проверки** — список параметров, наличие которых проверяется у отфильтрованных элементов.

  > [!WARNING]
  > Вручную задать параметры с нуля не получится, если только вы не понимаете структуру JSON-настроек. Поле сделано редактируемым, чтобы удобно копировать настройки параметров между правилами.

  Чтобы задать параметры для проверки, нажмите кнопку с тремя точками справа от поля — откроется [окно настройки параметров](#окно-настройки-параметров-для-проверки).

## Окно настройки фильтра

<img width="603" alt="Окно настройки фильтра" src="https://github.com/user-attachments/assets/3a25c661-f5a1-4cfa-824c-ca0beb0f3587" />

Окно поделено на 3 части:

1. Область, в которой задаём категории элементов для проверки в этом правиле.
2. Область выбора логики фильтрации между категориями и параметрами:
   - **Только категории** — в коллекцию попадут элементы выбранных категорий, параметрические условия игнорируются.
   - **Только параметры** — в коллекцию попадут элементы, прошедшие фильтрацию по параметрам, условия категорий игнорируются.
   - **Категории ИЛИ Параметры** — попадут элементы с совпадением хотя бы по одному критерию. ***Если у элемента совпадают оба критерия, он тоже попадёт в коллекцию.***
   - **Категории И Параметры** — попадут только элементы, у которых совпадают оба критерия.
3. Область создания параметрических условий. Рассмотрим её подробнее.

При старте в ней есть 2 кнопки и комбобокс для выбора логического оператора.

Кнопка **«Добавить простое условие»** создаёт одно простое условие.

<img width="580" alt="Простое условие" src="https://github.com/user-attachments/assets/60b33717-7e12-400c-b7db-6c74aa7a551a" />

В данном случае пройдут элементы, у которых *Имя параметра = значение*.

Кнопка **«Добавить групповое условие»** создаёт группу, внутри которой можно создать несколько простых условий. Внутри группы между всеми параметрами действует логический оператор **И**.

<img width="575" alt="Групповое условие" src="https://github.com/user-attachments/assets/0bb40bdc-14d8-4bb3-b6a8-d87b4454de83" />

В данном случае попадут только те элементы, у которых истинны оба условия.

В комбобоксе внизу области выбираем логическое взаимодействие между условиями (простыми и групповыми): **И** — нужно, чтобы у элемента совпали все условия; **ИЛИ** — элемент попадёт в коллекцию, если совпадёт хотя бы одно условие.

Сложный пример:

<img width="601" alt="Сложный пример условий" src="https://github.com/user-attachments/assets/c7ac1587-ddb6-420b-b25c-94e651cb4ab6" />

Здесь 2 простых условия и 1 групповое (внутри ещё 2 простых). Логика между условиями — **ИЛИ**. Чтобы элемент попал в коллекцию, достаточно, чтобы выполнялось хотя бы одно: *параметр 1 больше 2* **либо** *параметр 2 меньше или равен 4* **либо** *параметр 3 содержит 7, а параметр 4 существует у элемента*. Если совпадений больше — элемент, естественно, тоже попадёт.

Чтобы применить настроенное условие, нажмите **«Применить»** внизу окна. Сложные условия преобразуются в JSON и попадут в текстовое поле, откуда их можно скопировать в другое правило.

<img width="604" alt="Окно настройки параметров" src="https://github.com/user-attachments/assets/f5aada92-1996-44a9-b545-6347fed45065" />

Простое окно: по кнопке **«Добавить параметр»** добавляем текстовые поля, в которые записываем параметры для проверки у отфильтрованных элементов. При нажатии **«Применить»** параметры преобразуются в JSON и попадают в текстовое поле.

## Работа плагина

После нажатия **«Старт»** запускается процесс экспорта отчёта. Программа перебирает профили: если чекбокс профиля активен — заходит в него и циклически перебирает модели. Из каждой модели создаёт отчёт по всем правилам; если в настройках включено обновление сводного отчёта — обновляет его и переходит к следующей модели. Когда модели в профиле закончатся, переходит к следующему профилю. Когда закончатся профили — выдаёт сообщение об успехе, а свежевыгруженные отчёты будут ждать вас на рабочем столе.
